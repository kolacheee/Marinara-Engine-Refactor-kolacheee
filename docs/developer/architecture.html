Architecture

The current tree is layered by ownership, not by convenience.

Marinara is a Tauri desktop app with a React feature tree, a React-free TypeScript product engine, typed frontend Tauri adapters, thin Rust commands, and focused Rust capability crates. The current feature tree is physically split into catalog, runtime, modes, and shell, with dependency direction enforced by .dependency-cruiser.cjs.

Layer Current packages Allowed role Must not do
features/catalog agents, characters, chat-presets, chats, connections, gallery, knowledge, lorebooks, personas, presets. Own resource hooks, storage-facing types, query keys, and library/editor surfaces. Import runtime systems, mode UI, or shell tools.
features/runtime generation, haptics, tracker, visuals, world-state. Own shared runtime systems used by modes and shell. Import concrete modes or app-shell workflows.
features/modes conversation, roleplay, game, game-assets, router, shared/chat-ui, shared/scene-ui. Own concrete user modes, shared mode UI, and the mode composition point. Import shell tools, or import sibling concrete modes outside modes/router.
features/shell bot-browser, connections, imports, mari, notifications, onboarding, settings, spotify. Own app-level panels, modals, native integration UI, top-level reset/import workflows, and standalone Mari. Own mode product rules or duplicate engine orchestration.
The enforced direction is shell -> modes -> runtime -> catalog. Same-layer callers should still use public owner files instead of reaching into another package's private folders.

Actual Feature Owners

Area Owns in the current tree Important notes
catalog/chats Chat records, messages, folders, notes, memories, branches, swipes, prompt preview hooks, and chat query keys. It is a data owner. Shared chat rendering lives under modes/shared/chat-ui.
catalog/characters Character library hooks, groups, sprites, start-chat helpers, and character shell surfaces. Current public callers also use this package for persona data hooks and persona groups.
catalog/personas Persona panel/editor/modals and persona query-key file. This package currently has no index.ts; most shared persona hook consumers import from catalog/characters/index.ts.
catalog/connections Connection rows, folders, tests, types, and query keys. Connection panel/editor UI belongs to features/shell/connections.
catalog/agents, gallery, knowledge, lorebooks, presets, chat-presets Agent/tool records, image gallery hooks, knowledge source hooks, lorebook data/editors, prompt presets, and chat preset selection. Catalog code may call shared APIs and engine contracts/services, but not runtime or mode internals.
runtime/generation useGenerate, which binds React state, storage/LLM/integration adapters, and engine generation services. This is the main UI-to-engine generation adapter layer.
runtime/world-state World-state API, tracker snapshot bridge, controller hooks, display helpers, patching helpers, and the world-state store. Used by mode surfaces and tracker UI; do not move game turn rules here.
runtime/visuals Sprite overlay, pinned image overlay, weather effects, world info display, sprite placement, sprite display modes, and shared visual types. Visual display contracts are shared here instead of copied across modes.
runtime/tracker and runtime/haptics Tracker sidebar/panels/model hooks and the haptic capability hook. Tracker is runtime UI over world-state data; haptics calls integration commands through shared API transport.
modes/shared/chat-ui Shared transcript, input, settings drawer, overlays, gallery/files drawers, prompt peek, quick switchers, summaries, swipes, and setup wizard UI. It is shared mode UI, not generic app UI. It may use mode-neutral lower layers and catalog/runtime public APIs, but must not import concrete mode packages.
modes/conversation Conversation surface, view/input/message components, recent chats, connection gate, home FAQ, and autonomous conversation hooks. Conversation can call engine/modes/chat services; it must not import roleplay or game internals.
modes/roleplay Roleplay surface, scene hook, encounter hook, CYOA choices, context injection, HUD panels, roleplay tracker UI, and roleplay tracker API. Roleplay scene and encounter behavior call engine/modes/roleplay services.
modes/game Game surface, setup, party turns, generation, scene analysis, game API facade, stores, parser/display helpers, maps, combat, inventory, journal, checkpoints, and audio. Game product rules stay under engine/modes/game; the feature package binds those services to UI and shared APIs.
modes/router ModeSurface, the active mode composition point. This file is currently broad: it also handles active chat loading, pagination, swipes, overlays, shared generation actions, background/visual state, TTS hooks, and mode switching. New product behavior should move to the owning mode/runtime/catalog module instead of making the router broader.
modes/game-assets Managed game asset browser and editor UI. It is a mode-adjacent tool composed by the app shell through shell.ts.
features/shell/* Settings, provider connection management UI, admin data reset, Mari, SillyTavern imports, bot browser, onboarding, notification bubbles, and Spotify controls. Shell may coordinate top-level workflows, but storage formats, prompt behavior, provider transport, and native IO still belong lower.

Dependency Guard

Guard What it enforces
No circular dependenciesModule graphs must remain acyclic.
Shared stays lowersrc/shared cannot import src/features or src/app.
Engine stays framework and host independentsrc/engine cannot import React, Zustand, @tauri-apps/api, or src/shared/api.
Feature layers only flow downwardcatalog cannot import higher feature layers; runtime cannot import modes or shell; modes cannot import shell.
Concrete modes are separatedConversation/chat, roleplay, and game cannot import each other's concrete UI or engine mode internals. features/modes/router is the composition exception for React surfaces.
Shared mode UI stays lower than concrete modesfeatures/modes/shared cannot import concrete conversation, roleplay, or game packages.
Feature internals are privateCross-package imports into another package's components, hooks, stores, state, lib, api, or encounter folders are blocked.

Desktop Host Scope

The implemented host is Tauri desktop. The React app and TypeScript engine are reusable in principle, but the current concrete frontend transport assumes Tauri invoke, Tauri channels, Tauri file URL conversion, and desktop plugins. A future Docker/web host should be added as a parallel host adapter under src/shared/api and Rust HTTP facades, not as scattered browser checks inside feature components.

When Adding Or Moving Code

  1. Start with the owner: catalog data, runtime system, concrete mode, shell workflow, shared UI/helper, engine service, shared API adapter, or Rust capability.
  2. Use a package public entrypoint when crossing feature package boundaries.
  3. Move reusable mode UI to features/modes/shared; move reusable runtime systems to features/runtime; move generic UI to src/shared/components.
  4. Move product behavior to the owning engine mode or generation layer, not into React components.
  5. Move privileged local work to Rust commands/crates and expose a narrow shared API wrapper.
  6. Run pnpm check:architecture when imports or package structure change, and pnpm check:docs when these docs change.

If a convenient import crosses ownership, add or adjust a public owner contract instead of reaching into a private folder.
